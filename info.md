# Project File Information

This document provides an overview of the files and directories within this project.

## Root Directory

*   `.git/`: Git version control system directory.
    *   `description`: Brief description of the repository.
    *   `hooks/`: Scripts to customize Git behavior.
    *   `info/`: Contains global excludes and other information.
    *   `refs/`: Stores pointers to commits (branches and tags).
    *   `objects/`: Stores all data objects.
    *   `COMMIT_EDITMSG`: Last commit message.
    *   `logs/`: History of branches and other references.
    *   `HEAD`: Points to the current branch.
    *   `config`: Repository-specific configuration options.
    *   `FETCH_HEAD`: Records the tip of the last fetch from a remote repository.
    *   `ORIG_HEAD`: Reference to the HEAD before an operation that modified it.
    *   `index`: Git staging area.
*   `.gitignore`: Specifies intentionally untracked files to ignore.
*   `pyproject.toml`: Project configuration file (e.g., for poetry, setuptools).
*   `.python-version`: Specifies the Python version for tools like `pyenv`.
*   `README.md`: Project README file, providing general information about the project.
*   `.venv/`: Python virtual environment directory.
    *   `CACHEDIR.TAG`: Marks the directory as a cache.
    *   `.gitignore`: Specifies files to ignore within the virtual environment.
    *   `bin/`: Executables for the virtual environment.
    *   `pyvenv.cfg`: Configuration file for the virtual environment.
    *   `lib/`: Python packages and modules for the virtual environment.
    *   `lib64/`: (Symlink) Python packages and modules for the virtual environment.
    *   `.lock`: Lock file for the virtual environment.
*   `uv.lock`: Lock file generated by the `uv` package manager, ensuring consistent dependencies.
*   `.env`: Environment variables file.
*   `__pycache__/`: Directory for Python bytecode cache files, improving application startup time.
    *   `get_files_info.cpython-313.pyc`: Compiled bytecode for `get_files_info.py`.
    *   `prompts.cpython-313.pyc`: Compiled bytecode for `prompts.py`.
    *   `call_function.cpython-313.pyc`: Compiled bytecode for `call_function.py`.
*   `functions/`: Directory containing various helper functions.
    *   `__pycache__/`: Python bytecode cache for functions.
    *   `path_validation.py`: This file provides utility functions for validating file paths. `validate_path` checks if a given directory exists and is within the permitted working directory. `validate_path_lite` performs a lighter validation, ensuring the path is within the working directory without checking for existence.
    *   `get_files_info.py`: This file contains the `get_files_info` function, which lists files and directories within a specified path relative to the working directory. It returns a string with file names, sizes, and whether they are directories. It also defines `schema_get_files_info`, a `FunctionDeclaration` for the `get_files_info` function, used by the Generative AI model.
    *   `get_file_content.py`: This file contains the `get_file_content` function, which reads and returns the content of a specified file. It uses `path_validation` to ensure the file is within the permitted working directory. It also defines `schema_get_file_content` for the Generative AI model.
    *   `run_python_file.py`: This file contains the `run_python_file` function, which executes a specified Python script. It uses `path_validation` to ensure the script is within the permitted working directory and captures its output. It also defines `schema_run_python_file` for the Generative AI model.
    *   `write_file.py`: This file contains the `write_file` function, which writes or overwrites content to a specified file. It uses `path_validation` to ensure the file path is valid and within the permitted working directory, creating directories if necessary. It also defines `schema_write_file` for the Generative AI model.
*   `test_get_files_info.py`: This file contains unit tests for the `get_files_info` function, which is responsible for listing files and their information. It tests the function with different directory paths, including the current directory, a subdirectory, and a non-existent directory.
*   `test_get_file_content.py`: This file contains unit tests for the `get_file_content` function. It tests reading content from existing files, a file in a subdirectory, and a non-existent file, as well as attempting to access a file outside the allowed working directory.
*   `test_write_file.py`: This file contains unit tests for the `write_file` function. It tests writing content to existing files, creating new files in subdirectories, and attempting to write to a directory outside the allowed working directory.
*   `test_run_python_file.py`: This file contains unit tests for the `run_python_file` function. It tests executing Python files with and without arguments, executing a test file, attempting to execute a file outside the working directory, and attempting to execute a non-existent file or a non-Python file.
*   `prompt.txt`: A text file likely containing a prompt or template.
*   `main.py`: This is the main application file. It sets up an argparse for command-line arguments, loads API keys, initializes the Generative AI client, manages the conversation history, calls the AI model, and processes function calls detected in the model's responses. It handles verbose output and manages the overall flow of the chatbot interaction.
*   `prompts.py`: This file defines the `system_prompt` variable, which contains the initial instructions and role definition for the AI coding agent. It outlines the agent's capabilities, such as listing files, reading file contents, executing Python files, and writing files, along with guidelines for path handling and error resolution.
*   `call_function.py`: This file defines the `available_functions` object (a `types.Tools` instance) which describes all the functions the AI agent can call, along with their schemas. It also contains the `call_function` method which acts as an entry point for the AI to call tools and maps the function call to the relevant python file in the `functions/` directory.
